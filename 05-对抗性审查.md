# 对抗性审查：五路互搏的结论与处置

> 状态：v1（2026-07-03）。5 个独立攻击 agent（定位/技术/名录/架构/事实核查）对前四份文档发起共 55 个攻击点，其中 P0 级 15 个。本文记录全部攻击的判定与处置；标「已修」的已回填进对应文档，标「待办」的进入 v1.1 返工清单。总裁决：**五路一致认为方案该继续，无一路主张放弃；但定位论证需要重写、架构规范需要大改后才能开工。**

## 一、总裁决（先说结论）

1. **立论层站得住。** 五路攻击后依然无恙的核心主张：「声部而非人」的封装单位（被评为全套文档最坚固的第一性推导）；负空间作为原创维度（GPT-4o 仿海明威词汇丰富度反向超标是硬证据，被攻击的是「一票否决」这个实现而非负空间本身）；判别器+拒绝采样逼近极值点的机制；翻译问题的三条架构决定；法律定位框架；三大关键文献引用经联网复核全部准确。
2. **定位论证被打出三个 P0。** 最重的一刀：整套「可评测性」论证只对 L1（文体指纹）成立，而价值主张押在 L2（方法论复用）上——L2 场景里语言换了、计量指纹失效、「像不像」盲测在设计上不成立。其次：没有一个具名用户画像和付费意愿证据；「厂商目标函数不允许」的差异化表述会被一个反例击穿。
3. **架构规范被定性为「资产形态愿景，不是可施工规范」。** 缺 token 预算、缺语料工程步骤、缺测试集与负样本的物理位置、缺成本模型；生成 spec 与判别 rubric 同源导致结构性自我共谋；自举闭环无收敛保证。
4. **事实底盘大部分扎实，但错在承重点上。** 抽查 15+ 项，核心文献引用全对；错的四处（DSH 句长张冠李戴、VS 倍数夸大 50%、金庸案法院写错、Catch-Me 论文适用域偷换）恰好都在最容易被复核的位置。全部已修正。

## 二、P0 级硬伤与处置

### 1. L2 的验收地基缺失（定位路，最重的一刀）

攻击：三层判别器测的全是 L1 的「像不像」；用马丁的 POV 制度写中文故事时，stylometry 失效、盲测不成立，L2 的评测退回项目自己宣判死刑的「写得好不好」。可评测的那层（L1）自认不是价值最高的，价值最高的那层（L2）失去了评测地基。

处置（已修 02）：结构层/方法论复用的验收改为**决策规则命中率**——生成文本是否在每个决策点执行了 spec 里的规则，逐条核查、类似单元测试；同时把 L2 的承诺文案从「价值最高」降为「价值假说，验收方式为规则命中率+人工评审，保真等价性不承诺」。

### 2. 生成 spec 与判别 rubric 同源共谋（技术路）

攻击：spec 和 rubric 出自同一次深读、同一套三段式、同一个基座——生成器按 spec 写、判别器按 spec 的镜像打分，是按自己的 checklist 自查，结构上必然高分。防 hack 四件套只写在调研文档里，架构一条没吸收。

处置（已修 02）：rubric 强制独立蒸馏（不同模型或至少不读 spec 的独立 agent），两者只在人类仲裁下合并；**判别器上岗前先「判别判别器」**——在 held-out 真迹、同代作家、已知劣仿三类样本上测精度，达标才准进迭代循环；四件套逐条写进架构。

### 3. 负空间一票否决会杀掉真迹和 L3（技术+事实两路同时命中）

攻击：《受戒》的情感高点「要！」带感叹号，按露馅清单汪曾祺声部会把汪曾祺代表作的高潮句判死；海明威《Indian Camp》结尾就是直接的内心陈述——「绝不」在事实层面是假的。硬否决机制还在结构上杀死「恰当背叛惯例」的 L3 候选：文档一边把 L3 寄托于严判别器，一边设计了一个专杀 L3 的判别器。

处置（已修 01/02）：露馅清单从布尔禁令改为**频率/密度约束**（阈值直接取自真迹计量基线），只有真迹中严格零出现的模式保留硬否决；清单上岗前必须在真迹全集上回放自校验，误杀真迹的条款回炉；L3「可被捕获」的表述降级为开放问题，设想中的 L3 通道（豁免语言层计量、只过中高层判别、人类终审）标注为未验证。

### 4. 差异化论证会被一个反例击穿（定位路）

攻击：V0 跑在 Claude（对齐模型）上——如果均值回归是硬约束它同样约束我们；如果 prompt 工程能克服它厂商也能。「目标函数不允许他们做」不成立，真实差异是「愿意做策展劳动+承担法律暴露」。文献引用还存在命题偷换：证据证明的是「默认输出多样性坍缩」，立论需要的是「强条件下无法逼近极值点」——后者只有间接证据。

处置（已修 01/04）：差异化重写为诚实版——均值回归解释「默认输出为什么平庸」，不解释「厂商做不到」；可辩护的差异只有两条：每声部的深度策展与判别测试集（承认可复制、只是没人做），和差距归因闭环（调研确认全行业无先例）。04 加命题边界声明。

### 5. 没有用户、没有判决线（定位路）

攻击：五份文档没有一个具名用户画像；Sudowrite 的证据指向「像我自己」而非「像张爱玲」；汪曾祺露馅清单去掉名字功能不损失；公版剪刀差——能卖的（史记、张岱）没人要，有人要的（张爱玲、金庸）2046 年前不能卖。且 V0 验收标准在开工前已经向下漂移过一次，没有 kill criterion 的项目会无限自我续命。

处置（已修 01）：新增「用户与场景」小节，标明当前全部场景为**能力倒推、未经需求验证**，试点阶段加需求验证关卡（5-10 个真实写作者访谈：愿意为「像张爱玲的改写」付钱，还是只为「改得更好」付钱）；写死判决线（见 01 第十节）；商业形态三选一（纯研究开源/公版+译者声部产品/遗产授权）列为待拍板的定位级问题。

### 6. 架构缺四个结构性组件（架构路）

攻击与处置（均已修 02）：
- **token 预算**：红楼级封装 spec+范例轻松破 50k-100k token，「像技能包一样加载」塞不下 → SKILL.md 增加载清单（六场景各自声明加载哪些层+预算上限），spec 拆「决策规则压缩版」（供注入）与「证据全文版」（供审阅）两个渲染物。
- **语料工程**：底本拍板、污染剔除、一致性检验原来不在流水线里 → 插入显式「语料工程」步骤，产出 corpus-lock 文件。
- **判别器资产没有物理位置**：目录里没有测试集/负样本/阈值校准 → judge/ 增 testset（held-out 真迹，永不进 spec）、negatives（同时代对照+历史仿作+自产仿作）、calibration（分长度档置信度）。
- **试点选错**：史记同时撞上文言计量未验证、对照组难建、验收目标含混、盲测读者难找四个最难问题 → 试点换为鲁迅·小说 + 《红楼梦》 + 老舍·小说；史记降为方法论稳定后的压力测试。

### 7. 名录的数字系统不可信（名录+事实两路）

攻击：「第一批约 40」实数 56；张爱玲、金庸这两个最贵的声部恰好违反自家演化轴原则（不锁时期会蒸出均值糊）；同一作家跨版图指纹差 2 级；P0 占 33% 等于没有优先级；版权列三种计算口径并存、无司法辖区维度（川端在日本 2042、海明威在美国按出版+95 均未公版，而原文蒸馏恰恰发生在原文法域）；「按卷抽样」会把网文声部的核心卖点（全书尺度的伏笔账本）变成空头支票。

处置（已修 03 关键项，其余进 v1.1）：首批口径修正并加波次标签；张爱玲锁《传奇》期、金庸锁中期（射雕三部曲至天龙）；版权列加双法域警告并修正已知错误；结构型声部改「双轨语料」（语言层可抽样，结构层全文过结构标注管线、封装进账本而非原文）；跨表一致性仲裁、P0 两轴重定级列入 v1.1。

## 四、经攻击后站得住的东西（五路的共识）

- 声部单位、负空间维度、判别器闭环机制、翻译三决定、法律框架——五路全部主动确认。
- 名录路特别确认：网文版图「蒸结构不蒸句法」是全名录最有原创性的判断；三试点的第一性逻辑攻不动（架构路换掉史记是执行修正，不是逻辑否定）；各版图观察的版本学/译本缠绕坑清单「是真正读过材料的人才写得出的」。
- 事实路确认：立论三篇关键文献（RLHF 低熵聚类、best-of-N 反超、写手趋同实验）转述准确，「负空间是最大露馅点」的 47.79 vs 34.62 核实无误，stylellm 描述完全准确。

## 五、留给决策者的三个开放问题（文档无法自答）

1. **商业形态三选一**：纯研究/开源声誉项目？对外只做「公版+译者声部」产品（傅雷体、朱生豪体、和合本是原译双公版且对今天写作者有用的交集）？还是遗产授权路线（BBC 阿加莎模式）主攻版权期高价值声部？三条路的封装规格不同，不能都要。
2. **判决线数值**：01 第十节已写入判决线框架（试点封装相对 naive style prompt 基线无显著优势即判负转向），具体阈值需在试点开工前锁死，不留给未来漂移。
3. **盲测网络**：「熟读该作家的读者」是整个验收体系的最终锚点，也是未估算成本的稀缺资源。为 3 个试点组建盲测团是 V0 必须解决的非技术问题。
