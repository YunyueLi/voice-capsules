# 研究与竞品调研：学术、开源、产品、法律全景

> 状态：v1（2026-07-03）。由 8 路联网调研 agent 的结果合成，关键结论均带来源。结尾的「对方案的修订清单」是本文档最重要的部分——调研直接改变了架构文档里的若干判断。

## 一、核心假设验证：「巨匠=极值点，后训练拉向均值」不是修辞，是有实证的机制

这是整个项目的立论，调研结论：**成立，且有多篇同行评审论文从不同层面证实。**

**命题边界声明（对抗审查后补）**：以下文献证实的是命题 A——「对齐模型默认输出的多样性坍缩」；本项目立论还需要命题 B——「强条件约束（spec+负空间+拒绝采样）下仍无法保真复刻极值点」。B 目前只有 DSH 一篇间接证据，且它测的是裸 prompt、没测本项目管线。所以 A 解释「默认输出为什么平庸」，B 是 V0 要亲自验证的实验假设，两者不能混用。

- RLHF 对齐显著压低输出多样性、把输出推向少数「吸引子状态」：对齐模型的 token 熵更低、输出在 embedding 空间明显聚类（《Creativity Has Left the Chat》，[arXiv:2406.05587](https://arxiv.org/abs/2406.05587)）。
- ICLR 2024 的系统对照：RLHF 换来分布外泛化，代价是同一输入下的多样性被大幅压缩；关键旁证是 **best-of-N 采样在摘要任务上效果反超 RLHF**——推理期用判别器选样，能拿到 RLHF 拿不到的东西（[arXiv:2310.06452](https://arxiv.org/html/2310.06452v2)）。这直接支持我们「拒绝采样推向分布尾部」的机制设计。
- 最扎实的人类对照实验：38 名有写作/编辑经验的 Upwork 自由职业者，在独立写作、base GPT-3 辅助、InstructGPT 辅助三个条件下共写 300 篇议论文，结果**用对齐模型的人写出的文章互相趋同，用 base 模型的不会**，且归因分析证明同质化来自模型贡献的文本本身（[arXiv:2309.05196](https://arxiv.org/html/2309.05196v3)，ICLR 2024）。注意局限：议论文不是文学创作，向小说场景的迁移是外推。
- 根因定位：mode collapse 来自偏好数据的「典型性偏置」——标注者系统性偏好熟悉的、base 模型高似然的文本。重要推论：**极值分布的信息仍潜伏在权重里，只是被压抑，可以被采样策略部分逼出来**（Verbalized Sampling，[arXiv:2510.01171](https://arxiv.org/abs/2510.01171)，创意写作任务上多样性较直接 prompt 提升 1.6-2.1 倍）。注意这只是方向性支持：VS 恢复的是「多样性」，不等于能恢复到某位特定作家的极值分布。
- 后训练同时压缩小说的主题、情感、文体三维变异（Narrative Flattening，[arXiv:2605.27878](https://arxiv.org/pdf/2605.27878)）；LLM 创作的不确定性显著低于职业作家（arXiv:2602.16162）。
- **诚实标注**：也存在反对「LLM 必然同质化」的论文（Ghods 等），说明结论依赖度量口径——这恰好加强了「必须自建可复核判别器、不能引用单一结论」的立场。

## 二、LLM 模仿名家的现状：表层能对齐，深层稳定露馅

- 最贴合本项目的实证（牛津 DSH 2025，[GPT-4o 模仿海明威/玛丽·雪莱的计量分析](https://academic.oup.com/dsh/article/40/2/587/8118784)）：句长层面仿作接近真迹（真海明威 14.15 词/句，GPT-4o 仿作 12.95-13.47），但随机森林判别器（LIWC 特征）以 84% 准确率区分仿作与真迹，真迹「始终形成独立聚类」。最漂亮的破绽：**GPT 版海明威的词汇丰富度 47.79，真海明威只有 34.62——方向完全相反**。模型不知道海明威的贫瘠是刻意的。这是「负空间是最大露馅点」的最硬证据（数字经对抗审查联网复核）。
- 大规模评测（Catch Me If You Can，[arXiv:2509.14543](https://arxiv.org/html/2509.14543v1)）：6 个 SOTA 模型模仿 400+ 作者，**few-shot 从 2 个示例加到 10 个几乎无增益**，非正式文体（博客）归属准确率崩到两位数，默认回落「通用平均腔」。**适用域注意（对抗审查后补）**：该研究的对象是 everyday authors（普通人的博客、邮件）——弱指纹、不在预训练语料里成体系存在；文学巨匠恰恰相反（全集大概率在预训练语料中、指纹 5 级）。「巨匠场景的 in-context 天花板在哪」是待验证问题，不能直接外推。
- 角色扮演研究线（Character-LLM、RoleLLM、CharacterEval、InCharacter、CoSER）做的是「答得像这个人」而非「写得像这个声部」，但有两条可搬：**CoSER 证明真实文学语料远胜合成问答数据、且「内心独白」是保真的关键变量**（[arXiv:2502.09082](https://arxiv.org/html/2502.09082)）；InCharacter 的「访谈式探针评测」思路可迁移成风格探针。
- 中文社区实验（腾讯新闻「去 AI 味大作战」仿鲁迅）给出可操作参数：in-context 范文 0k 时套路、**2k 字明显改善、10k 以上边际递减**。范例库的规模起点由此定。另一个佐证：李继刚「汉语新解」prompt 仿鲁迅杂文腔能传播出圈——杂文声部比小说声部好仿，印证声部要拆开、难度分级。

## 三、判别器可行性：技术成熟，但四个陷阱必须设防

三档判别器路线全部有现成肩膀可站：

1. **计量层**：Burrows/Cosine Delta + 功能词 + 字符 n-gram。工具现成（[stylo](https://github.com/computationalstylistics/stylo)、[faststylometry](https://github.com/fastdatascience/faststylometry) 的距离转概率范式）。中文要点：**字符级优先**，分词是噪声源；文言虚词表（《红楼梦》前 80 回/后 40 回判别研究用「之、其、亦」等 50 多个虚词 + SVM）是现成种子。最有判别力的特征选型：Author Multilevel N-gram Profiles + 词嵌入（DSH 论文）、burstiness（节奏起伏，Nature 系刊 O'Sullivan 研究，AI 文本系统性偏低）。
2. **风格嵌入层**：对比学习的作者表示（STAR/PART/LUAR/[StyleDistance](https://arxiv.org/html/2410.12757)），同作者拉近、异作者推远，短文本比 Delta 强。
3. **LLM 判别层**：LIP/PromptAV 式的语言学引导 prompt（GPT-4 作者验证达 84-92%，[arXiv:2403.08213](https://arxiv.org/html/2403.08213v1)），价值在**可读的差距归因**——这是自举循环的反馈信号。

四个必须设防的陷阱：

- **长度约束**：传统方法可靠判别需约 5000 词，低于 1000 词显著劣化（Eder 2015）。判别器要按文本长度分档给置信度，短文本降级到嵌入相似度。
- **内容泄漏**：判别器最大陷阱是把题材当风格（「鲁迅=写旧中国」）。必须用同题材改写平行样本控制内容变量（StyleDistance 专门解决此问题；PAN 用同人小说跨宇宙评测就是为了逼出纯文体信号）。
- **reward hacking**：判别器一旦被当奖励优化，生成器会学出「骗过判别器但不像巨匠」的捷径（奖励模型爱抓长度/格式等伪特征有专门研究）。对策：held-out 判别器、多判别器投票、对偏离幅度加 proximity 正则（RBoN，[arXiv:2404.01054](https://arxiv.org/html/2404.01054v1)）、人类专家定期校准。
- **对抗性处境**：刻意模仿能把归属器打到接近随机（Adversarial Stylometry，模仿成功率 67%）。我们主动模仿巨匠=天然对抗设定，判别器要把自己生成的仿作当 hard negative 持续回灌加固。

另一个清醒剂：市面的「AI 文本检测器」检测的是当前 LLM 的均值化痕迹（.95+ 精度），我们的目标恰是消除这种痕迹——它们是我们要打败的裁判，不是可采信的裁判。

## 四、工程先例：离我们最近的几块积木

- **[stylellm_models](https://github.com/stylellm/stylellm_models)**：中文最接近的先例——基于 Yi-6b 分别微调《三国》《西游》《水浒》《红楼》四个文风改写模型（**注意：只有这四部，没有金庸/鲁迅模型**；鲁迅另有独立项目 [LuXun-GPT](https://github.com/Suffoquer-fang/LuXun-GPT)）。它验证了「以作品为声部」可行，其「通用模型出内容 × 文风模型转写」两段式管线是内容-风格解耦的现成雏形。缺的正是我们要做的三件事：判别器、差距归因、迭代闭环——它只有样例展示、零定量评测。
- **StyleAdaptedLM**（[arXiv:2507.18294](https://arxiv.org/html/2507.18294v1)）：最值得直接抄的范式——LoRA 在非结构化风格语料上学风格、与指令模型权重合并，**用作者归属分类器 F1（0.90-1.00）当风格保真指标**，同时保住指令跟随。这基本就是我们「封装 LoRA + 判别器验收」的可行性证明。
- **AuthorMix**（arXiv:2603.23069）：每作者一个 LoRA、免梯度分层混合——「一声部一适配器、可组合可插值」有学术验证，将来「海明威句法 × 金庸场面」的杂交实验有现成技术路径。
- **ZeroStylus**（arXiv:2505.07888）：精心设计的免训练模板抽取对长文风格迁移只有边际提升（6.90 对 6.70）——纯 prompt 路线天花板的直接证据。
- **语料**：英文公版全集用 SPGC（5 万+书带作者元数据）和 PG-19 现成切分；中文现当代作家不在其中且多有版权，需自建并留合规台账。

## 五、产品全景：没有人在做「特定极值点的保真复刻」

| 产品 | 风格能力 | 对我们的教训 |
|---|---|---|
| Sudowrite（Muse 模型） | Match My Style：用户 2000 词样本→自动生成风格描述 prompt，注入位置在指令最后、权重最高；Muse 是只用授权小说训练的专用模型 | 「样本→风格描述」是最低成本路径，**应作为我们的 baseline 对照组**：任何封装打不过一段好的 style prompt 就没有存在价值 |
| NovelAI（AI Modules） | 产品形态最接近我们：每个作家风格是独立可插拔的训练模块（预置爱伦·坡、奥斯汀、洛夫克拉夫特），社区有模块仓库生态 | **最大的反面教训：风格资产绑死基座权重，报废发生了两次**——自定义模块仅支持 2021-22 代模型，2023 年 Clio/Kayra 一代即不再兼容，2024-10 官方以基础设施为由停止模块训练（旧模块仅存活在 legacy 模型上）。封装本体必须是基座无关资产，权重只能是可再生缓存。另一层教训在需求侧：作家模块存在多年未成为付费驱动点、停训后社区反弹有限——**作家级保真的付费需求在所有已知先例中均未被证实，本项目须在试点阶段自行验证** |
| 彩云小梦 | 题材/流派级续写（言情/玄幻），日产 4 亿字、MAU 200 万 | 流派均值满足的是娱乐消费；C 端为「风格」付费的是陪伴玩法不是文学保真——我们的用户更可能在创作者端 |
| 阅文·妙笔 | 网文行业模型，IP 知识问答+素材生成 | 一线作家实测划出能力分界线：「词汇比我大、场景更丰富，但掌控不了节奏和爽点」——**篇章层是保真的真难点**，封装的结构层/读者契约层不能虚写 |
| Weaver/蛙蛙写作 | 1.8B-34B 创意写作垂直基座模型，对齐「职业作家偏好」 | 两条教训：自建垂直基座资本上跑不通（2024-10 被 OPPO 收购）；「对齐作家偏好」本质是造一个更高级的新均值，仍不是极值点 |
| character.ai 作家 bot | UGC 人设卡，无逐 bot 训练 | 人设扮演≠文体保真：莎士比亚 bot 说的是莎翁的事、用的是平台的腔。知识底座与语言风格必须分维建模 |
| OpenAI creative writing 模型（2025-03 预告） | Altman 晒 AI 短篇自称被击中 | 反面教材：作家反应两极——Winterson 在 Guardian 撰文赞赏，Eggers 斥为「pastiche garbage」，教授数出堆砌隐喻，金句与纳博科夫《Pnin》撞句。正因为没有判别器，赞弹双方都只能各执印象——**没有可复核判别证据就晒样本，恰好反证判别器的必要性**；每个声部发布必须附判别分数与盲测数据 |
| DeepMind Dramatron | 层级 prompt 链生成剧本，15 位行业专家结构化评估、剧本真实上演 | 评测方法论值得抄：找真正的专业人士做结构化验收，而非只跑自动指标 |

Claude 顺带一提：作家群体口碑里「AI 腔最轻、最听风格指令」的通用模型，多位出版作者实测其模仿自己文风成功——这是我们 V0 执行底座的现实依据。

## 六、前车之鉴：先人都死在哪一层

- **Botnik《哈利·波特》仿写章节**（2017，病毒式出圈）：词汇层可仿，叙事/情节必崩；出圈靠喜剧定位+重度人工策展，恰恰放弃了保真。
- **LSTM 续写《冰与火之歌》**（2017）：死人复活、凭空造人——长程一致性和知识底座层的失败。
- **南京大学 AI 续写《红楼梦》后四十回**（2024）：负责人直言产出「幼稚、空洞的正确废话、套路格式」；最大障碍是跨学科——懂文学的不懂技术、懂技术的不懂文学。我们的对策：文学侧深度参与判别 rubric 制定。
- **微软小冰诗集**（2017）：学 519 位诗人、27 个马甲投稿骗过读者和编辑，仍被诗人集体差评「平庸、无此在」。**最重要的验收启示：「骗过普通读者」和「专家认可」是两道门槛，必须分档验收。**
- **星新一奖 AI 小说**（2016，过盲审首轮）：真相是人定情节角色、AI 组装——结构层可控、人物层是公认短板。
- **产品化雷区实例**：Grammarly「Expert Review」因以名家口吻给反馈被集体诉讼（2026-03）；BBC 的 AI 阿加莎·克里斯蒂写作课走了遗产授权路线；个人产品 Ghost Writer 被测评指出连风格认错作家——反衬「判别器保证认得出是谁」是产品底线。

## 七、法律边界：安全区清晰，雷区也清晰

- **风格本身不受版权保护**（中美同源的思想-表达二分法；美国 17 U.S.C. §102(b) 及判例明确风格/方法不受保护）。「提炼风格规范+训练判别器」处在最大的法律安全区，这应该作为项目的定位宣示。
- **但边界在金庸诉江南案**（广州知识产权法院 2023 年 5 月二审终审改判）：单个人物名不受保护，**60 多个人物的「人物群像」结构（名称+性格+关系网）被认定为受保护的表达**，构成侵权；判赔 188 万元，且法院不判停止侵权、改判再版按版税 30% 付经济补偿——后一点对本项目是有利信号（人物群像侵权成立时法院仍倾向经济补偿而非下架）。含义：蒸馏停在文体/技法层安全，打包搬运某作家的角色宇宙和情节骨架会踩线。
- **训练语料**：Bartz v. Anthropic（2025）裁定合法购得图书训练=变革性合理使用，但盗版语料导致 15 亿美元和解——**语料来源合法是不可谈判底线，每个声部要有语料合法性台账**。Authors Guild v. OpenAI 未终局，最高风险点是「输出与原作实质相似」（记忆回吐）——判别器应兼任回吐检测器。
- **人格权**：《民法典》994 条保护死者姓名/名誉等人格利益，近亲属可诉；2024 年 AI「复活」明星事件被官方定性为不论营利与否均侵权。含义：**对外产品化时去人格化命名**（「某某声部/文体」而非「张爱玲写的」），不冒名、不暗示背书。
- **公版速查**（中国大陆=死后 50 年，人身权永久）：鲁迅（1936 卒）、老舍（1966）、周作人（1967）已公版；沈从文（1988→2039）、张爱玲（1995→2046）、汪曾祺（1997→2048）、钱钟书（1998→2049）、金庸（2018→2069）均未公版。**个人研究场景中美都有合理使用空间、风险低；对外产品化则数据来源、输出回吐、署名冒用三条线同时收紧**——版权内作家限内部研究，对外先上公版声部。
- **法域适用（对抗审查后补）**：版权按使用地法律算，而架构决定「外国作家语言层从原文蒸馏、产出用于该语言写作」——即语料获取和使用可能发生在原文法域。两个典型：川端康成在中国大陆 2023 年起公版，但在日本受死后 70 年保护至 2042 年底；海明威在美国按「出版+95 年」逐作品计算（《老人与海》到 2048 年）。名录的版权列需拆成「大陆公版年+原文主要法域状态」双字段（v1.1 待办），外语声部建档前按原文法域重新过一遍。

## 八、风格形式化：维度清单可以站在百年学术肩膀上

架构文档的七层模型可以逐层挂靠成熟框架，评审者认：

- 语言层 ← Leech & Short《Style in Fiction》的四大类清单（词汇/语法/修辞格/衔接与语境——文体学标准打钩表）+ Biber 多维分析的可量化特征（[Neurobiber](https://arxiv.org/pdf/2502.18590) 已做成快速抽取器，白盒特征便于差距归因）。
- 叙事层 ← 热奈特《叙事话语》三轴（时间/语式/语态：时序、时距、频率、聚焦、叙述层），且已被 Piper/So/Bamman（EMNLP 2021）操作化映射到 NLP 任务。
- 结构层 ← 计算叙事学的事件化情节模型；2026 年有专门综述（arXiv:2602.15851，引用前核对）。
- 风格→结构化向量的工程先例 ← LISA（EMNLP 2023）：LLM 把风格标注成 130 万条自然语言属性再压成 768 维可读向量——「spec→可评测维度」的流水线直接可抄。
- 评分框架 ← microsoft/LLM-Rubric（ACL 2024）的多维+校准范式。
- **中国文论（风骨/文气/气韵）概念齐备但没有可操作化成果**——「文气」要自建代理指标（句群语势/停顿密度/虚实节奏），不能声称有现成框架。
- **负空间没有任何现成学术框架**——这是我们的原创点，也是调研反复证实的最大露馅维度（海明威词汇丰富度反向、AI 仿写「加戏内心独白」「用力过猛」）。

## 九、对方案的修订清单（调研改变了什么）

1. **纯 prompt 路线的天花板可能比预想低（假设，非结论）**。ZeroStylus 的边际提升、Catch-Me 的 few-shot 无增益提示 in-context 有天花板——但注意 Catch-Me 测的是弱指纹的日常作者，对预训练语料中高频出现的巨匠是否成立未知（见第二节适用域注意）。因此处置为：V0 的任务是把判别器闭环跑通、拿到「结构化 spec vs naive style prompt」的对照数据，**这本身就是检验该假设的实验**；微调是否提前为 V1 主路线（限公版声部），等三个试点的 prompt 基线分数出来再定，不在立项文档预支结论。
2. **封装必须基座无关**（NovelAI 报废教训）：spec、范例、露馅清单、判别器、测试集是本体资产；任何权重是可再生缓存。架构文档已符合，升级为硬约束。
3. **验收分两档**：普通读者盲测（小冰能过）和专家认可（小冰没过）是两道门槛，声部发布标准写死「专家档」。
4. **判别器防 hack 四件套**进架构：held-out 判别器、多判别器投票、proximity 正则、专家定期校准；同题材平行样本防内容泄漏；自产仿作回灌做 hard negatives。
5. **范例库参数起点**：单场景 in-context 范文 2k-10k 字区间做 A/B（中文仿鲁迅实验的经验值）。
6. **法律护栏**进架构：语料合法性台账、回吐检测、对外去人格化命名、公版声部先行对外。
7. **护城河排序（对抗审查后校准）**：判别器与测试集是核心**工程**资产（没有它闭环不存在），但不是可防御的**商业**资产——原料公开、方法可复制、无独立买家。真正难复制的是两样：差距归因闭环的先发数据积累（每次迭代产生的露馅点数据库）和专家盲测网络本身。「差距归因」环节在所有已调研对象中均无先例，是真正的原创点，也是最该先做穿的一段。
